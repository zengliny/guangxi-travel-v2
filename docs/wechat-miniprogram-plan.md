## 一、需求分析

### 1.1 目标用户画像

| 用户类型 | 核心需求 | 痛点 |
|----------|----------|------|
| **北海游客** | 景点导览、民宿预订、行程规划 | 信息分散、预订繁琐 |
| **养老买家** | 候鸟养老房源、气候环境、医疗配套 | 缺乏可信渠道 |
| **亲子家庭** | 亲子景点、安全民宿、遛娃攻略 | 找不到合适的 |

| 优先级 | 功能模块 | 描述 | 复杂度 |

## 二、技术选型

### 2.1 跨端框架对比

| 维度 | 原生开发 | UniApp | Taro |
|------|----------|--------|------|
| **学习成本** | 高（需分别学微信/抖音） | 中（Vue/UniApp） | 中（React/Vue） |
| **代码复用** | 0% | 80%+（一套代码多端） | 90%+ |
| **生态成熟度** | ★★★★★ | ★★★★☆ | ★★★★☆ |
| **性能** | ★★★★★ | ★★★★☆ | ★★★★☆ |
| **组件库** | 微信自带 | uView、UniUI | Taro UI |
| **打包体积** | 最小 | 中等 | 中等 |
| **社区活跃度** | 官方 | 活跃 | 活跃 |
| **适合项目** | 简单功能 | **推荐：中小型项目** | 复杂大型项目 |

### 2.2 推荐方案：UniApp

**理由：**
- ✅ 一套代码同时输出微信小程序 + 抖音小程序 + H5
- ✅ Vue 语法，上手快，社区生态成熟
- ✅ 性能足够，满足当前业务需求
- ✅ 开发效率高，维护成本低

**备选方案：**
- 若后续需做 App → 升级到 uni-app x
- 若团队熟悉 React → 考虑 Taro

### 2.3 后端服务方案

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **自建服务器** | 完全可控、数据自主 | 需运维、成本高 | 大型项目 |
| **云开发 (微信/抖音)** | 免运维、免费额度够 | 依赖平台、扩展有限 | **推荐：中小型 MVP** |
| **Serverless (Vercel/Netlify)** | 免运维、按量付费 | 冷启动、限制 | 轻量 API |
| **小程序云开发** | 免服务器、数据库集成 | 需学习新API、限制 | **推荐：MVP 快速上线** |

### 2.4 推荐方案：小程序云开发

**架构：**
```
┌─────────────────────────────────────────────┐
│                微信/抖音小程序               │
├─────────────────────────────────────────────┤
│              uni-app 前端                    │
├─────────────────────────────────────────────┤
│           云开发 - 云函数                    │
│    (用户认证、支付回调、业务逻辑)             │
├─────────────────────────────────────────────┤
│           云开发 - 云数据库                  │
│    (民宿、景点、订单、用户收藏)               │
├─────────────────────────────────────────────┤
│           云开发 - 云存储                    │
│    (图片、视频、用户头像)                     │
└─────────────────────────────────────────────┘
```

**理由：**
- ✅ 免费额度：云函数 100万次/月、数据库 2GB、存储 5GB
- ✅ 免服务器运维，专注业务开发
- ✅ 天然支持微信登录、支付
- ✅ 快速上线，适合 MVP 验证

## 四、API 设计

### 4.1 API 规范

- 协议: HTTPS
- 认证: JWT Token (云函数自动校验)
- 格式: JSON
- 编码: UTF-8

### 4.2 接口列表

| 模块 | 方法 | 路径 | 描述 |
|------|------|------|------|
| **民宿** | GET | /homestay/list | 民宿列表 |
| | GET | /homestay/detail | 民宿详情 |
| | GET | /homestay/rooms | 房型列表 |
| **景点** | GET | /destination/list | 景点列表 |
| | GET | /destination/detail | 景点详情 |
| **订单** | POST | /order/create | 创建订单 |
| | GET | /order/list | 订单列表 |
| | GET | /order/detail | 订单详情 |
| | POST | /order/cancel | 取消订单 |
| **用户** | POST | /user/login | 微信登录 |
| | GET | /user/profile | 获取用户信息 |
| | PUT | /user/profile | 更新用户信息 |
| **收藏** | POST | /favorite/add | 添加收藏 |
| | DELETE | /favorite/remove | 取消收藏 |
| | GET | /favorite/list | 收藏列表 |
| **咨询** | POST | /message/send | 发送消息 |
| | GET | /message/list | 消息历史 |
| **支付** | POST | /payment/create | 创建支付 |
| | POST | /payment/callback | 支付回调 |

## 七、风险与建议

### 7.1 潜在风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 微信审核驳回 | 延迟上线 | 提前熟悉规范，避免诱导分享 |
| 支付资质 | 无法收款 | 需注册商户号（个体工商户即可） |
| 数据同步 | 多端数据不一致 | 统一云开发，数据实时同步 |
| 地图配额 | 超额收费 | 申请配额，先用免费额度测试 |

### 7.2 建议

1. **先做 H5 版**：快速验证业务，再发布小程序
2. **数据复用**：现有网站数据可直接迁移到云数据库
3. **分步发布**：先上民宿+景点，后续迭代订单+客服
4. **监控很重要**：接入小程序数据分析，追踪用户行为
